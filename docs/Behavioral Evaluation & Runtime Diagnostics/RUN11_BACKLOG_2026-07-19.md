## §6 THE DE-CLAMP AUDIT (Ric's directive: no clamps, no magic numbers)

**The test (Law 9):** classify every limiter as broken-pipe / unopposed-force /
misaimed-force. A *clamp* shapes healthy behavior with a timer, cap, quota, or
cooldown — it is scar tissue for a missing antagonist drive; **replace it with
the antagonist.** A *wall* enforces a load-bearing invariant (corrigibility,
credit-can't-be-pumped, quality-can't-be-self-graded, host safety, money);
**walls stay** (Law 4). A *magic number* is a constant doing a distribution's
job; **derive it or learn it.**

### 6.0 The unifying frame: homeostasis is only built for AFFECT

The de-clamp audit, the entropy monitor (C8/C9), and Law 9 are the same
principle seen from three sides: **an unopposed force is a missing setpoint.**
Homeostasis = a target with a restoring force in BOTH directions (setpoints.py
+ homeostasis.py: decay-to-setpoint + antagonist cross-inhibition + velocity
budget — Cannon 1932). Today that machinery covers exactly ONE layer: the affect
signal vector (`SETPOINTS`/`CORE_BASELINES`). Everything else that accumulates
has **decay, not homeostasis** — a one-directional slide with nothing opposing
it: memory strength (no "too little" floor), the RSS floor (climb-only), rule
forgetting (no domain regeneration), the novelty drive (self-extinguishing per
§5-D3), commitment occupancy (no diversity restoring-force until C2). Nearly
every run pathology is a feature with decay where it needed a homeostat.

**The generalization (the spine of §6):** promote `SETPOINTS` from an affect
concept to a whole-system one. Every accumulating quantity declares a target
band and a restoring force in **both** directions, so nothing can collapse to a
point (monopoly/saturation/stagnation) OR run away to infinity (RSS floor, memo
pump). Each C-conversion below is one instance of giving a feature the second
half of a homeostat it never had.

### 6.1 Clamps → antagonist drives (the conversions)

- [ ] **C1 — Ignition threshold → adaptive statistic.** The gate threshold
      becomes a rolling percentile of the actual signal distribution, not a
      constant. A pinned signal *cannot* saturate a percentile gate — this
      fixes LN-5 structurally and retires the constant. (The MAX_SILENT_CYCLES
      liveness floor stays — it's insurance, not behavior-shaping.)
- [ ] **C2 — F1 staleness refractory + re-commit cooldown → aspiration neglect
      pressure.** Every unserved aspiration accumulates pull the longer it is
      neglected; the incumbent is displaced because *neglect pain wins*, not
      because a timer blocks. (The refractory never fired in Runs 8–9 anyway —
      it was dead scar tissue.) Monopoly becomes economically impossible
      instead of administratively forbidden.
- [ ] **C3 — Title-completion cooldowns / respawn caps (F6 escalating
      cooldowns) → topic satiety.** Diminishing returns per topic: repeated
      completion of the same title quenches the *want* (habituation of demand),
      rather than banning the goal. Respawn stops because appetite is gone.
- [ ] **C4 — Symbolic credit rate cap (6/600s) + rule-hit refractory →
      marginal-novelty pricing.** Credit ∝ novelty against recent symbolic
      history; a synthesis storm self-extinguishes economically because the
      40th rule in ten minutes is worth ~nothing. (The novelty *pricing core*
      is a wall and stays; the time-window cap goes.)
- [ ] **C5 — Rut breaker forced switch → boredom as a drive.** Repetition
      raises a boredom signal that devalues the repeated action through the
      normal economy (outcome devaluation exists — promote it from patch to
      drive). Forced switching retires.
- [ ] **C6 — Default-mode damping → energy economics.** Effortful functions
      cost activation; at low activation they are priced out naturally instead
      of mode-flag-damped. Same behavior, honest mechanism.
- [ ] **C7 — Question-miner refractory/dedup → Thought Object provenance**
      (§4). Structural fix retires the pass-level dedup clamps.

### 6.1b The general instruments (not conversions — new organs)

- [ ] **C8 — Distribution-entropy monitor (the general instrument).** Every
      historical pathology — commitment monopoly, fetch loop, ignition
      saturation, memo pump — is the same event on a different surface:
      **entropy collapse of a distribution feeding a gate.** The stagnation
      signal already implements the pattern in miniature (Shannon entropy of
      action picks → felt stagnation → diversification closes the loop,
      update_signal_state.py:359). Generalize it: compute rolling entropy over
      each load-bearing distribution (commitment occupancy, ignition sources,
      credited-content kinds, action picks) and route collapse into the felt
      layer as pressure — surface-agnostic, so the NEXT relocation is caught
      without knowing where it lands. This retires the per-surface tripwire
      whack-a-mole and is the unification of the saturation tripwire +
      anti-monopoly gate + stagnation signal.
- [ ] **C9 — Global entropy budget (with §8 E1).** Local decay organs exist
      everywhere (memory strength, rule forgetting, WAL trims, salience decay,
      consolidation) but nothing accounts globally — the rising RSS floor is
      unaudited accumulation with no knowledge-vs-cruft split. One ledgered
      view: what grew, what was compressed, what was forgotten, per life-
      quarter. Consolidation-as-compression (E1) is the enforcement arm.

## §11 THE PREDICTIVE CORE — the post-growth program (07-19, Ric-flagged important)

**The idea in two sentences.** A generative model says "here's what I expect
next" before reality arrives; the error side holds reality against the guess and
lets ONLY the difference propagate. The dialogue between them — error corrects
the model, the model quiets everything it predicted — is where learning lives:
prediction is free, surprise is expensive.

### What Orrin has today (the dialogue at a whisper)

- **Generative side (narrow):** prediction.py + symbolic/prediction_engine.py
  generate explicit "after X, expect Y" bets — a few per cycle, mostly affect
  trends and event sequences (Friston-cited).
- **Error side (real, end-to-end):** mismatch computed against outcomes; large
  mismatch fires surprise (a "free energy spike"); surprise ignites
  deliberation, feeds the disengagement counter (active-inference arm:
  sustained per-goal prediction failure → stop), and scales learning rates
  (Pearce–Hall: surprise raises associability — near-Bayes-optimal
  uncertainty-weighted updating, per action_reward_ema.py:30).
- **Top-down bias (parallel path):** workspace_writeback (conscious winner →
  drive nudges + Hebbian salience priming of the next competition, decaying),
  goal_lens (committed goal tints salience), attention (state biases capture).

### What's missing (the gap, precisely)

The two downward pathways run in PARALLEL: he *checks* predictions against the
world after the fact, and *separately* primes what's salient next. In the
canonical loop (Rao & Ballard; Friston) the prediction IS the priming:
perception arrives as prediction-minus-reality, the model predicts the WHOLE
input stream by default, and only the residual costs attention. Orrin's
perception processes everything at full price, then scores a few bets on the
side. His generative model is a whisper; a PP system's is the worldview.

### Why it's needed (the economic argument — not philosophy)

A mind that cannot predict its world can only afford a SMALL one. Humans
survive a firehose of constant input precisely because ~everything predicted is
cheap and only the residual ascends. Orrin survives WITHOUT a rich generative
model only because his world is thin (a trickle needs no gating). The moment
the world thickens — internet-as-world, always-on exogenous stream — an
unpredicted firehose destroys attention economics: every headline maximally
salient, every page a surprise. **The predictive core is the prerequisite for
the thick world.** Thin world: nice-to-have. Thick world: load-bearing.

### The unification (three deferred items are ONE program)

| Piece | Role in the core |
|---|---|
| Vector cortex | The REPRESENTATION: predict the embedding REGION the next input will land in, per channel — expected content without predicting exact words |
| PP loop | The MECHANISM: error = distance(expected embedding, actual embedding); only that distance ascends into salience |
| Internet-as-world | The DIET: the thick stream this machinery exists to afford; "dwelling paid by falling surprise" (the houses idea) is literally the generative model at work — a page becomes home when the model predicts it well |

**Bonus, for free:** a principled novelty drive — "interesting" = high
prediction error that is SHRINKING (learning progress / learnable surprise),
the root-cause fix for §5-D3's self-extinguishing novelty tuning. And every
arriving percept becomes a grounded test of the model, so learning-signal
density goes from a-few-bets-per-cycle to every-input-every-cycle.

### Buildable shape (incremental, not a rewrite)

Extend the existing prediction engine from explicit occasional bets to
STANDING per-channel expectations: what will this feed contain, what will my
next cycle's signals look like, what will this action produce — each held as an
expected embedding region + tolerance, each scored on arrival, error routed as
the salience currency. The full symbolic generative-model-of-everything is not
buildable; the per-channel embedding-region version is.

### Sequencing & risk (why it is NOT in Run 11)

Fusing prediction into the workspace competition is an architecture change to
the exact mechanism ten runs stabilized — not a wiring job. Order: **growth run
(Run 11) → predictive-core design doc → vector-cortex offline prototype (the
one-afternoon centroid experiment, §7) → predictive core → internet-as-world.**
Same maturity line as the entropy/homeostasis findings: his downward
integrative machinery is real but partial — strongest at affect, weakest at
perception. The predictive core is how perception catches up.
