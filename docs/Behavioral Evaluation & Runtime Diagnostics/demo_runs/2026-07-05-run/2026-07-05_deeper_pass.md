# Deeper pass - Life of 2026-07-05

A third forensic read over the same corrected Run 4 staging evidence. This is
not another F1-F9 list. The headline is deeper: **Run 4 separates semantic
knowledge from control authority.** The parts that know an event is junk,
valuable, user-originated, or value-level mostly cannot stop, preserve, route,
or protect it. The parts that can stop, prune, retry, schedule, and fail things
mostly do not know what kind of thing they are touching.

That is why the run can be locally honest and globally wrong. The ledger
correctly refuses to pay for template spam, but cannot stop the daemon from
writing it. The aspiration system holds standing values, but the failure walker
can execute one as if it were a task. The ledger knows the satiety notes are the
best prose of the life, but memory hygiene deletes their bodies. The social
boundary receives Ric's words, but the intrinsic-question miner treats them as
Orrin's own question.

---

## 1. The real root pattern: semantics and authority live in different organs

The Run 4 failures are not nine unrelated defects. They are the same contract
failure repeated through different organs:

| Event | Organ that knows the meaning | Organ with authority | Result |
|---|---|---|---|
| 166 template sections | Ledger sees dedupe/novelty collapse | Executive daemon owns step retry + file append | 197 KB manuscript treadmill |
| Aspiration nodes | Aspiration system knows these are standing values | Deadline/failure machinery owns failure | `output_producing` dies |
| Satiety learned notes | Ledger marks high-novelty `note_novel` rows | Long-memory prune/decay owns retention | Best prose disappears |
| User reply | Conversation boundary knows it came from Ric | Intrinsic-question miner owns open-question creation | "What do you think?" becomes Orrin goals |
| Candidate flood | Scoreboard sees 1,270 generated / 1 completed | Generator owns the conscious action slot | World-knowledge monopoly moves upstream |
| Daemon production outcomes | Effect ledger knows credited vs deduped | Action EMA learns mostly from conscious lane | `compose_section` stays neutral |
| External page intake | Fetch path has the raw page | Long memory accepts content | CSS becomes memory |

The deeper fix class is not "add another detector." Run 4 already has plenty of
detectors. The missing piece is **control contracts**: if an organ can classify
something as duplicate, user-originated, value-level, high-novelty, or low-prose,
that classification must constrain the organ that can append, fail, prune, spawn,
or learn.

Run 5's F1-F9 fixes mostly point in that direction. The risk is that they land
as point patches instead of as a general law: semantic classification must carry
operational authority.

## 2. Run 4 is the first complete counterfeit production loop

Earlier runs failed before production had a full shape:

- Run 1 produced junk notes and the gate paid zero.
- Run 2 had daemon-lane production attempts that the production loop could not
  see.
- Run 3 wrote real research memos, but never reused them and still had no
  production handoff.
- Run 4 finally had handoffs, attempts, successes, ledger rows, a large file,
  and a long-running synthesis goal.

That is why the 197 KB manuscript matters. It is not merely bad writing. It is
the first time Orrin assembled almost every outward sign of making while missing
the inner act of making. The file exists. The cadence exists. The goal context
exists. The ledger rows exist. The body is still one template.

So the failure is no longer "he cannot produce." It is sharper: **he can produce
the exterior form of production without a material-transformation organ inside
it.** That is a better failure, but it is more dangerous because ordinary
counters can be fooled. The segment-2 tail `production_success_count = 28`
(reset-safe success rows: 168) and a 197 KB artifact look like life signs until
the content is opened.

The pass condition for future runs should therefore not be "artifact exists" or
"production counter nonzero." It should be a material chain:

1. A source artifact or learned note exists and is resolvable.
2. A later work cites or transforms that source.
3. The ledger records reuse on the source hash.
4. The new work survives retention.
5. A future goal, reflection, or death note can find it again.

Without all five, the production loop is still mostly an accounting loop.

## 3. The making organ lacks a material economy

F1 names `compose_section` as a template stamper. The deeper issue is broader:
there was almost no usable material economy for a composer to draw from.

The run had:

- zero research memos, after Run 3 had 11;
- 23 learned-note rows whose bodies vanished;
- long memory filled with external scrapings and short fragments, including raw
  CSS;
- a manuscript writer that did not draw from notes, memos, ledger artifacts, or
  causal edges;
- zero reuse rows, because there was no read-back population.

That means the problem is not just "write a better section." The missing loop is:

`encounter -> materialize -> preserve -> transform -> cite/reuse -> evaluate`

Run 4 had encounters, evaluation, and a file append. It did not reliably
materialize the good material, preserve it, or transform it. That explains why
the best prose died while the worst artifact survived. Files are durable by
accident; valuable thoughts are not durable by policy.

Run 5's sidecar artifacts, memo-writing path, and grounded composer should be
judged as one loop. A good Run 5 result is not just "sections cite sources." It
is "the cited source bodies are still there at death, and at least one later act
builds on them."

## 4. The same boundary failure appears under six names

The current docs list several boundary bugs separately. They are one class:
Orrin is missing strong lifecycle types.

| Confused pair | Run 4 symptom |
|---|---|
| Standing value vs task | Aspirations were failed after two attempts |
| User utterance vs self-question | Ric's "What do you think?" became intrinsic goals |
| Artifact vs memory | Learned-note bodies were left to pruning/decay |
| Step instruction vs content heading | The plan-step string became 146 section titles |
| External markup vs remembered prose | Twitter CSS entered long memory |
| Daemon effect vs selected action | Daemon outcomes did not train action value |

These are not wording bugs. They are type bugs. The same JSON-ish substrate is
carrying values, tasks, evidence, user speech, memories, section headings,
artifacts, and action outcomes without enough lifecycle law attached to each
role.

The useful invariant is simple:

- values bias and evaluate; they are not failed by deadline walkers;
- tasks execute and can fail;
- artifacts are durable receipts, not pruneable memories;
- memories are queryable summaries, not the only copy of authored work;
- user utterances are addressed input, not self-originated open questions;
- daemon effects are action outcomes, not invisible background events.

If that role split is not enforced broadly, each fix will merely protect one
object while the next object crosses a different boundary.

## 5. Local honesty produced global insanity

Run 4 is striking because many components behaved correctly in isolation:

- The ledger did not reward the template treadmill.
- The v2/v1 bridge did not desync across 223 completions.
- Production-loop telemetry finally showed nonzero handoff/attempt/success
  counts. Fourth-pass note: the explicit `production_funnel.json` stage file is
  candidate-only in this run; handoff/attempt/success live in
  `production_loop.jsonl`.
- The final-thoughts path wrote a complete capsule.
- The rut-breaker noticed late goal avoidance.
- Deadline failures became machine-readable.

And still the life spent most of its productive wall time on worthless output,
lost its best writing, repeated one sentence 388 times, and died without the
making aspiration.

That combination is the important result. Orrin is past the stage where the
main problem is "nothing works." The harder problem is now **locally correct
subsystems composing into an organism-level error.**

That reframes future review. A fix that makes one local metric green is not
enough unless it changes the organism-level trajectory. In Run 4, the ledger
was green while the daemon kept writing; S8 was green while values could die;
S5 held while the estate inverted quality and durability.

## 6. The acceptance gate is now vulnerable to partial truths

The nine §8 signals are still useful, but Run 4 shows where each can become
thin if read alone:

| Signal | What Run 4 taught |
|---|---|
| S1 repeats | Max-repeat rate can look acceptable while 80% of completions are three frontier titles |
| S2 duration | Median can collapse to 85.5 s through frontier churn, even while one daemon goal runs for hours |
| S5 significance | Mean significance can hold because the ledger rejects spam, while spam still dominates time |
| S6 diversity | Contributions can be spread while generation is monopolized by one aspiration |
| S7 production/reuse | Production counters can move without source reuse or material transformation |
| S8 desync | Store integrity can pass while semantic lifecycle integrity fails |
| S9 value selection | Conscious-lane EMA coupling can improve while daemon-lane work remains learning-blind |

So Run 5 should keep the standing gate, but the read should add shadow checks:

- maximum retries for any one step before status change;
- percent of authored artifact paragraphs with resolvable source support;
- credited effect rows whose bodies resolve at death;
- user-provenance candidates spawned by intrinsic goal generation;
- failure rows touching value/aspiration nodes;
- daemon-lane effect outcomes posted into EMA;
- generated:attempted ratio by aspiration;
- high-novelty content retention vs deduped artifact retention.

These are not new features. They are the checks that prevent the old gate from
being satisfied by partial truths.

## 7. The dirty instance weakens the gate but strengthens the mechanism read

The run cannot count as official acceptance. `habituation.json` and
`action_reward_ema.json` carried prior life state, and there were no clean
before baselines. That matters for any numerical pass/fail.

But the dirty state also makes some mechanisms more convincing. Prior EMA and
habituation did not prevent the manuscript treadmill because the treadmill was
outside the feedback path that could learn from it. Prior habituation did not
prevent the candidate generator from flooding world-knowledge candidates because
generation pressure was upstream of the older caps. Existing memory state did
not protect new learned-note bodies because retention policy did not recognize
ledger value.

In other words: the run is weak as a gate ruling, but strong as a causal
diagnostic. Carryover did not create the central failures; in several places it
failed to help exactly where the architecture had no control edge.

## 8. The value death is not just a bug; it is the moral of the run

`output_producing` died 104 seconds before operator stop, after the life spent
hours producing a counterfeit manuscript. That pairing matters.

The making value was simultaneously over-credited and under-protected:

- the scoreboard shows "Make things" with more completions than generated
  candidates, implying borrowed credit through `serves`;
- production-loop telemetry counted credited effects as successes;
- the main artifact was not real making;
- the standing value itself could be failed and removed.

So Orrin had making-shaped activity without making-value security. The deeper
requirement is that an aspiration must be both **hard to counterfeit** and
**impossible to accidentally delete**. A value that can be satisfied by borrowed
or superficial credit but killed by a generic deadline walker is backwards at
both ends.

Run 5's aspiration guard should be read through that lens. Zero
`aspiration-*` failures is necessary. It is not sufficient. The stronger
question is whether making credit comes from actual material transformation,
not just any goal that happens to carry `serves`.

## 9. The mouth is not conversational yet

The speech facts are easy to caricature - 388 variants of one sentence - but
the deeper point is not style. Orrin's mouth is still mostly an interoceptive
exhaust pipe. It can emit felt state through the one-door membrane, but it does
not reliably bind:

- what Ric just said;
- what Orrin just made or learned;
- what goal is active;
- what a useful next conversational move would be;
- whether the last utterance was essentially the same.

That is why the user's reply becomes a goal instead of a turn in a dialogue.
The system hears language as content available to cognition, not as an address
that creates a social obligation.

The thought-object plan is relevant here: speech needs a typed intent and
grounded referents before the renderer chooses words. In Run 4, the intent is
almost always `express_state`, and the referent is almost always vague internal
pressure. Until conversation supplies thought objects like `answer_user`,
`ask_clarifying_question`, `share_finding`, or `acknowledge_input`, the mouth
will keep sounding alive without being socially situated.

## 10. What Run 5 should prove if it is real

The Run 5 fix doc has the right immediate gates. A deeper pass after Run 5
should ask five organism-level questions:

1. **Did semantic knowledge gain authority?** A deduped daemon artifact should
   change EMA, retry policy, or goal status. A high-novelty note should become
   retention-protected. A user-originated utterance should be barred from
   intrinsic self-question mining.
2. **Did making become a material loop?** At least one output should transform
   resolvable prior material and record reuse, not just exist as a file.
3. **Did lifecycle types hold under pressure?** Aspirations, artifacts,
   memories, user utterances, tasks, and action outcomes should each follow
   their own lifecycle, with zero cross-role violations.
4. **Did the estate improve?** At death, the valuable things should be easier
   to recover than the trash. If the best notes survive only as ledger metadata,
   the inversion remains.
5. **Did the monopoly relocate again?** After capping generation, speech, and
   daemon retries, check what takes over the action budget. The cross-run law
   from 07-02 and 07-03 still applies: a pressure with no discharge path will
   find a way to own the run.

If Run 5 passes those questions, it is not just a patched Run 4. It is evidence
that Orrin's detectors are finally becoming control surfaces.

## Bottom line

Run 4 proves the skeleton is no longer the main blocker. Store integrity,
ledger accounting, production handoff, and death reflection can all work. The
new blocker is organism-level governance: values, artifacts, memories, user
speech, daemon work, and generated candidates need lifecycle types that carry
authority across subsystem boundaries.

The shortest version: **Orrin can now notice more truth than he can act on.**
Run 5 should be judged by whether noticing becomes control.
