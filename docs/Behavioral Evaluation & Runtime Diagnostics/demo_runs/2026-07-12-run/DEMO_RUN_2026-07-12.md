## Verdict: gate NOT passed — but this is the run that *isolated the cause*.

Run 7's job was to make credit un-pumpable. **It worked, completely.** The value
pump is dead (`self_understanding.value_ema` **0.5196** vs Run 6's poisoned
0.8142), the memo loop is gone (most-rewritten file **2×** vs 403×), the ledger
actively rejected junk (8 `duplicate` + 4 `low_significance` effect rejections),
and content-keyed credit did exactly what it promised: **the monopolist earned
almost nothing.** `self_understanding` held the committed slot for 90.9 % of life
and was credited **1** contribution; the actual producer, `output_producing`,
earned **7**. Credit no longer follows the slot-holder.

And the committed-goal monopoly **survived at 90.9 %** anyway
(`aspiration-self_understanding`, 10,052 / 11,060 cycles), with `stale_cycles`
**10,291** and `avoid_streak` **6,852**.

That is the whole result. For six runs the monopoly was defensible as a
credit/pump artifact — kill the pump and it goes away. Run 7 killed the pump and
it did not go away. **The monopoly is not a reward bug. It is structural**, and
Run 7 is the first run whose evidence points cleanly at the structure rather than
at another leaky credit path. The fix that follows from it —
`RUN8_FIX_PLAN_2026-07-14.md` — is aimed there, and this verdict adds one cause
the plan did not yet have (§4.2).

one per cycle) and `commitment_signals.json`.

  10,052 / 11,060 cycles.** 24 distinct goals *ever* held it; the incumbent was
  91 % of them.
- **106 driver transitions, none durable.** The longest *non-incumbent* hold was
  **324 cycles** (`g_de1a9baa65f4`, an early research goal). The only meaningful
  interlude was a cluster of "Understand X more deeply" research goals around

- **The penalties climbed unbounded and did nothing.** `stale_cycles` reached
  10,291 and `avoid_streak` 6,852 — but both saturate in `commit_score` at 120
  and 20 cycles respectively (a flat −30). From cycle ~120 onward the incumbent

- **The incumbent's plan was frozen.** `goals_mem.json`: its plan is a *single*

  as "version 5" **five times** between 23:58 and 00:35, never resolved. Last

- **Avoidance was recognized ~16,864 times** across the rotated activity logs,

**Crucially, `self_understanding` is the *only* goal in `commitment_signals.json`

## 4. Why it monopolized — two reinforcing structural causes

### 4.1 All commitment authority is *relative*, and there was no rival (RUN8 F1)

`commit_score` (`commitment_value.py:240`) can subtract at most `_W_STALE +
_W_AVOID = 30` from the incumbent, and both terms saturate early. That penalty
only matters if a *rival* is within 30 points and can overtake it. In Run 7 there
was no such rival, so the saturated −30 sat inert for ~10,000 cycles while the
counters climbed into five figures. There is **no absolute release** — nothing
that says "past this staleness the holder yields regardless of what can replace
it." This is exactly what `RUN8_FIX_PLAN` F1 adds (arm the dormant
`recommit_block_pulls` on an absolute stale trigger).

### 4.2 The directional rotation pool has exactly one member (NEW — feeds Run 8)

`order_committable` enforces "exactly ONE directional `long_term` goal drives at
a time, rotating on learned value/staleness/avoidance." The intent is rotation
*among directions*. But the eligibility test is
`tier == "long_term" and (g.get("directional") or g.get("never_complete"))`, and
in this life **only `self_understanding` carries those flags**:

```
self_understanding   directional=True  never_complete=True  frontier=True   contrib=1
world_knowledge      directional=None  never_complete=None  frontier=False  contrib=3
genuine_contact      directional=None  never_complete=None  frontier=False  contrib=0
output_producing     directional=None  never_complete=None  frontier=False  contrib=7
```

So the "rotate among directionals" machinery governs a **single-member set**.
`self_understanding` occupies the one directional slot by construction; the other
three aspirations are never *directions* that can drive — they only ever
contribute through ordinary sub-goals (which is why `output_producing` earned 7
contributions yet never held the slot). The asymmetry is a downstream effect of
the **causal-frontier-introspection** reframing: the causal graph is ~100 %
self-model, so the frontier-goal generator only ever produced a
`self_understanding` frontier (`_frontier_seq: 220`), and only that aspiration
got promoted to `directional`.

**Implication for Run 8:** F1 (absolute release) is necessary but, alone,
insufficient in spirit. When F1 forces `self_understanding` to yield, the slot
goes to an *ordinary goal*, not to another *direction* — because no other
direction exists. Run 8 should carry a companion fix: **flag all four aspirations
`directional` (or otherwise admit them to the directional pool)** so the driver
slot rotates among genuine directions, not merely "self_understanding vs. a
research chore." Recommend tracking this as **F2** in the Run 8 build. (It also
predicts a cleaner G1 pass: with four eligible directions and an absolute
release, no single one can hold > 60 %.)
