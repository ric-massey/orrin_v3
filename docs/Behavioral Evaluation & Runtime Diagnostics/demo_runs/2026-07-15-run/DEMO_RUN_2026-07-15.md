moved it to 42.6 %.** All four aspirations drove the slot; `genuine_contact` held

it is real in **both** runtime segments (54 % pre-crash, 44 % post-fix — both

But the cause is not the lever the gate scores. **F1 (the absolute staleness
refractory) never fired** — no goal ever went stale long enough to trip it,
because **F2 (admitting all four aspirations to the rotation pool) kept the slot
turning over so staleness never accumulated.** Prevention beat cure. The gate's
`G2` demands the release *actually fire* (`refractory_events` non-empty); it is
empty. So:

- **Objective: MET.** Monopoly broken, no thrash, no idleness, production up.
- **Formal gate (`Pass = G1 ∧ G2 ∧ G4`): NOT passed** — `G2` unmet (F1 never
  triggered), and `G5` regressed on reuse (2 < Run 7's 4).
- **F1 remains unproven in life.** The `ORRIN_STALE_REFRACTORY=0` ablation arm is
  now the priority, to confirm F2 is the operative lever (and whether F1 is a
  still-needed backstop or dead weight).

This life also **exposed and fixed a real crash** (`make_candidate` keyword-only
dispatch, §0) — the single most consequential engineering event of the run.

## 0. The crash (mid-life) — what the user flagged

The `silent_death` in `lifecycle_events.jsonl` was **not** a natural death: it
was an uncaught `TypeError` that killed the brain thread mid-**cycle 4418**
(the exact cycle missing at the `production_loop` seam, 4417→4419), after which
the process pulsed as a zombie for 6.5 h until the watchdog noticed. (The
"~4253" a first read of `crash.log` suggests is stale block-buffered stdout —
the `tee` buffering gotcha; the logger timestamp and the seam are authoritative.)

```
TypeError: make_candidate() missing 2 required keyword-only arguments:
           'kind' and 'direction'      (brain/loop/invoke.py:108, fn(**built))
```

`_invoke_cognition`'s dispatchability guard checked `POSITIONAL_ONLY` and
`POSITIONAL_OR_KEYWORD` params but not required `KEYWORD_ONLY` params, so a
function with `(*, kind, direction)` looked callable and blew up on call.
The **uncommitted fix** adds `KEYWORD_ONLY` to the guard; segment 2 then logged
*"make_candidate … not directly dispatchable; skipping"* and ran crash-free.

**Follow-ups:** (a) ~~commit the `invoke.py` fix~~ *(done — `e70ac98`)* — the
**regression test** (required-keyword-only cognitive function is skipped, not
dispatched) is still missing; (b) the watchdog took 6.5 h to notice a dead
cognitive loop while pulses continued — the liveness check keys on process
pulse, not cycle advance. Worth a shorter cycle-stall tripwire, and it should
key on `production_loop` cycle stamps: of the three counters in play at the
crash, the heartbeat lagged 43 s / 18 cycles and stdout lagged ~160 cycles;
only `production_loop` was accurate to the seam. Full trace:
`logs/CRASH_TRACEBACK.txt`.

## 3. Why the monopoly broke — F2 did it, F1 stood down

`RUN8_FIX_PLAN` diagnosed two reinforcing causes of the Run-7 monopoly:
(1) all commitment authority is *relative* and saturates with no rival → **F1**
(absolute release); (2) the directional rotation pool had exactly one member →
**F2** (admit all four aspirations).

The capture shows **cause (2) was the load-bearing one.** With four roughly
equal-value directions (`value_ema` 0.50–0.625) admitted to the pool, the driver
slot rotates every few cycles among them — so no single goal ever sits idle long
enough to accrue the `stale_cycles` (250) that would arm F1. F1's trip condition
was never reached; its machinery (`commitment_value.py:162-169`) is present and
correct but dormant all life.

This is a **better** outcome than the gate anticipated (prevention over forced
release — no clock-driven churn, `G4` never at risk), but it means F1 is
**untested in a live loop**. If a future run ever produces a genuine
single-occupant slot (e.g. three aspirations retired), we still do not know F1
fires. **The ablation arm is now mandatory, not optional.**
