# Orrin LLM Cognition Audit

Scope: `ric source code(1).zip`, searched for `generate_response`, `routed_response`, `gated_generate`, `llm_callable_by`, `llm_available`, `llm_ok`, and direct `llm(...)` hooks.

## Core gating

- `brain/utils/generate_response.py:97-112` — LLM-tool-only default. `ORRIN_LLM_TOOL_ONLY` defaults true. Only `ask_llm`, `ask_llm_for_research`, `ask_llm_about_conversation`, and `user_chat` are allowlisted.
- `brain/utils/generate_response.py:365-369` — if tool-only and caller is not allowlisted, returns `tool unavailable: llm`.
- `brain/utils/llm_gate.py:91-109` — `llm_callable_by(caller)` is the real cognition gate: LLM available + either tool-only disabled or caller allowlisted.
- `brain/utils/llm_gate.py:123-148` — names of cognitive functions that require LLM and are filtered from candidate pool unless cognition can call the LLM.
- `brain/utils/llm_gate.py:179-192` — `filter_llm_dependent` removes LLM-only cognition when `llm_enabled` is false or tool-only is true.

## Highest-impact cognition changes

1. Cognitive function selection: `brain/think/think_utils/select_function.py` uses `filter_llm_dependent`, so enabling full LLM cognition adds mental actions to the candidate pool.
2. Inner-loop deliberation: `brain/think/inner_loop.py` switches from `run_inner_loop_symbolic` to LLM draft/critique/revision/TOT paths.
3. Simulation: `brain/think/simulate.py` enables lookahead, branch simulation, debate voices, and judge synthesis.
4. Goal planning/execution: `brain/cognition/planning/goal_planning.py`, `goal_execution.py`, and `goals.py` replace symbolic planning/decomposition/fallback/completion checks with LLM paths.
5. Comprehension: `brain/cognition/comprehension.py` replaces fallback parsing with LLM JSON parse.
6. Intrinsic goals: `brain/cognition/intrinsic_goals.py` can generate richer internal goals from current mental material.
7. Autogenerated thoughts: `brain/cognition/self_generated/autogenerated_thoughts.py` switches from symbolic composition to LLM-written spontaneous thoughts.
8. Symbolic crystallization: `brain/symbolic/llm_gate.py` can call LLM after symbolic routing and crystallize successful answers back into symbolic rules/memory.

## Bottom line

With `llm_enabled=true` but `ORRIN_LLM_TOOL_ONLY=1`, most cognition paths above still do not reach the LLM. Orrin stays symbolic-first and can use the explicit LLM tool/user chat. With `ORRIN_LLM_TOOL_ONLY=0`, the listed paths can change actual cognition: planning, simulation, self-reflection, internal monologue, intrinsic goals, values/identity/selfhood, repair, experimentation, and self-extension.
