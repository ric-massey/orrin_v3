# Orrin Docs Index

Docs are grouped by track. Each track holds the **live** plans/audits at its top
level; superseded or completed docs move into that track's `archive/` (or the
shared `archive/` at the docs root). Run reports live under
`Behavioral Evaluation & Runtime Diagnostics/demo_runs/`, dated by run.

## Tracks

- **Core Architecture, Embodiment & Evolution/** — the master plan and embodiment
  design. Live: `MASTER_PLAN_2026-06-16.md`, `ALLOSTATIC_CAPACITY_TAX_2026-06-17.md`
  (design note, not yet built). Superseded sources in `archive/`.

- **Behavioral Evaluation & Runtime Diagnostics/** — what Orrin actually does at
  runtime: goal-system anatomy, the production-reward plan, signal→action audit,
  the life-capsule plan, plus all dated `demo_runs/`.

- **UI, Security & Desktop Packaging/** — single source of truth is
  `UI_SECURITY_DESKTOP_MASTER_PLAN_2026-06-16.md`. The four consolidated source
  docs (UI audit report + remediation plan, desktop-remaining, learning-visibility
  ideas) are read-only history in `archive/`.

- **Engineering & Code Health/** — non-behavioral engineering debt: the structure
  audit and the cleanup plan.

- **Capability, Benchmarks & Evidence/** — `BENCHMARKS.md` (run guide) and
  `CLAIMS_AND_EVIDENCE.md` (public claims vs. proven evidence).

- **Language & Cognition/** — `ORRIN_LANGUAGE_PLAN.md`, the human-level-language
  roadmap.

## Archive

- `archive/` (docs root) — completed/superseded plans, audits, and fix records
  from earlier in the project.
- Each track may also keep its own `archive/` for docs specific to it.
