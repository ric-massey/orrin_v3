## Part 2 — The finding that reframes the whole task

**The templates are training the native voice.** The native LM is live (103,023
steps, 39.7M tokens seen, 5.3M params, checkpoint `brain/data/language/native_lm.pt`
rewritten the same day). Asked to generate, it produces:

```
"My sense of something is positive — it came up in: ..."    ← opinions.py:745 template, verbatim
"Something is working through me that I haven't named yet."  ← autogenerated_thoughts.py:124, verbatim
"I looked through my own files" / "I found something I wanted to do"  ← acquisition.py _ACTION_PHRASES frames
```

The acquisition loop (`cognition/language/acquisition.py` → `native_lm.train_on`,
wired at `ORRIN_loop.py:2979/3000`, `dreaming/dream_cycle.py:422`, and
`narrate_experience` at `think/think_module.py:369`) feeds long memory, felt
narrative, opinions, and dialogue back into the organ. Because the symbolic
fallbacks **emit templates**, those templates are now the most-rehearsed phrases
in his corpus.

> Every template left in place is not a neutral placeholder. It is the sentence
> Orrin is learning to think in. Fixing the gate (`llm_available → llm_callable_by`)
> without replacing the template **promotes the template from "rarely fires" to
> "fires every cycle and trains the voice harder."**

Consequences:
- For functions whose fallback is a **template** (regret, autogenerated_thoughts,
  opinions, introspection-summary), the dead-fallback fix and template removal are
  **one job, not two** — do not flip the gate until the engine is built.
- For functions whose fallback is a **real engine** (goals, self_reflection), the
  gate fix alone is a genuine win — flip those now.
- **The native LM is not static — it demonstrably learns from use.** Feeding it
  real (non-template) work will grow it; feeding it templates is actively harmful.
  The growth mechanism is sound; the input is contaminated.

## 3. Risk assessment — what to watch after restart

1. **Symbolic lookahead is conservative by design** (vetoes only on a confident net-negative
   projection). Given the causal graph is currently sparse, expect it to almost always *proceed*
   — effectively the old honest-skip until beliefs accumulate. This is intentional; the failure
   mode it avoids (false vetoes → `think_more` loops) is the dangerous one. **Watch:** the
   `[meta_ctrl] symbolic lookahead` private-log line for any unexpected VETO clustering.
2. **Planner intent families are heuristic.** A goal whose verb is ambiguous (e.g. "research what's
   happening") may route to a neighbouring family. All outcomes are still *executable*; the risk is
   sub-optimal tool choice, not breakage. **Watch:** `[pursue_goal] Symbolic plan …` logs for
   tool/goal mismatches.
3. **`values_check` refusal is high-precision/low-recall** (pattern-based). It will *miss* subtle
   value-violations the LLM might have caught, by design (a false refusal is worse). **Watch:**
   refusal rate — should be near-zero on normal input.
4. **`opinions`/`regret`/`autogenerated_thoughts` now write engine output to memory**, which feeds
   the language corpus. Confirmed clean (no telemetry decimals, no canned phrases), but **watch**
   the native-LM corpus over a few consolidation cycles to confirm no new template-like recurrence.
5. **`behavior_generation`** now calls `goals._rule_based_decompose` (a cross-module private import).
   Low risk, but a refactor of that helper would silently affect this path.

## 4. Architectural notes / deliberate non-changes

- **`temporal_planner` is NOT wired into execution** — it emits non-executable belief-chains and is
  correct only for reasoning/lookahead. Wiring it into `pursue_goal` (as the original plan doc's #5
  literally said) would have broken goal execution. This was the single most important correction.
- **`decide_to_write_code` left on the `ask_llm` tool path** — the intended "Orrin invokes the tool"
  design; not converted, not filtered.
- **`#11 speech` and `#4 critique` untouched** — already symbolic-exclusive / symbolic in tool-only.

## 5. Rollback

All changes are source-only and uncommitted; `git checkout -- <file>` reverts any file. The Phase 1
data hygiene kept `.bak`/`.bak2` for `opinions.json`, `long_memory.json`, `replay_corpus.txt`.
No migrations, no schema changes, no irreversible writes in the code path.

The conversion is **safe to ship to the tool-only deployment.** The highest-judgement piece (the

lookahead is deliberately under-powered rather than over-powered. Recommend a **post-restart watch**
of the five log lines in §3 for one session before considering it settled.
