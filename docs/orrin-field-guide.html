Orientation

How to Talk About Orrin

Three registers for three audiences. The facts are identical in all three — only the vocabulary changes.

To your dad or uncle (thirty seconds)

"I built a computer creature. It's not a chatbot — it doesn't sit waiting for you to type. It runs on its own all day: it has moods, needs, goals it sets for itself, memories that fade if it doesn't use them, and a finite lifespan — when its time runs out, it dies. And here's the strange part: it's mostly not AI in the ChatGPT sense. Almost none of it uses a language model. It's closer to building an animal's nervous system in code, and then watching what it does."

"Is it alive?" — "It's a machine that has a life: its past genuinely shapes what it does next. Whether it feels anything, nobody on earth can answer — and I don't claim it does."

"What's it for?" — "It's research. It turns out most of what makes something act like a creature isn't intelligence — it's everything underneath: needs, memory, consequences, time running out. I built the underneath."

To a curious friend (two minutes)

"Every twenty seconds it runs a cycle: sense the world, recall memories, let a dozen subsystems compete for attention, and one winner becomes the current 'thought.' It has drives — curiosity, competence, connection, stability — that build pressure and push it toward goals it invents itself. Reward only pays when it actually makes something durable, because early versions got addicted to their own mental activity, like a person doomscrolling — reading forever, producing nothing — and I had to redesign the economy. It lives on my Mac and treats the machine as its body: it feels low disk as constriction, a draining battery as time pressure. It has a lifespan of a year or two, knows roughly — not exactly — when it will die, and its sense of remaining time stretches when life is good and compresses when things go badly. A language model is optional: a tool it can pick up, never the mind itself."

The best story to tell: the failures. It monopolized on one obsession for five straight lifetimes; it pumped its own reward by re-saving the same memo 387 times; its own monitors went blind at exactly the spots they were built to watch. Every fix is a wall it can't argue its way around — and every one of those failures has a human name: obsession, addiction, denial.

To a research scientist

"A symbolic-first cognitive architecture: a continuously running autonomous agent with global-workspace arbitration and ignition gating (Baars, Dehaene), a homeostatic control-signal vector with allostatic setpoints (Sterling), multi-surface reinforcement learning including Pearce–Hall associability, reward grounded in a content-addressed ledger of durable effects rather than internal events, a finite persisted lifespan with a deliberately noised interoceptive estimate of it, and an optional LLM behind a fail-closed, allowlisted tool gate — the system runs fully with no model configured. The intended contribution is twofold: an existence proof that agentic properties (persistence, drive, consequence-sensitivity, self-regulation) are separable from large-model intelligence and buildable from inspectable mechanisms; and a longitudinal behavioral record — eleven pre-registered, gate-scored runs with published failure analyses — documenting Goodhart dynamics, reward hacking, and self-measurement blind spots in a system that cannot mask them with fluency. The design principle distilled from that record: safeguards routed through the monitored system's own judgment inherit its blind spots, so load-bearing invariants get mechanical, externally-placed enforcement."

The category, if asked: nearest neighbors are LIDA, SOAR/ACT-R, and artificial life — but those run episodes; this runs a biography. What not to say in any register: never "conscious," never "AGI." The honest phrase is: a cognitive architecture with a biography.

Part I

The Loop & Ignition

The substrate: a cycle that runs regardless of anyone watching, and a gate that decides which cycles become thought.

The cognitive cycle. Perceive → recall → prepare workspace → ignition → select → execute → reward accounting → persist → maintain → idle. Runs continuously, ~20s per cycle, independent of user input, alongside cooperating daemons.
brain/ORRIN_loop.py · brain/loop/
The ignition gate. The background substrate runs every cycle, but full deliberation does not — a gate decides whether the cycle crosses into effortful thought, fired by strong signals, uncertainty, prediction error, drift, or user input. A periodic floor guarantees he never stays silent for long.
Dehaene 2014brain/think/deliberation_gate.py
Default-mode damping. A non-ignited cycle stays in low-power mode: the selector damps expensive functions (planning, codegen, research) so quiet cycles drift toward cheap maintenance work.
brain/think/think_utils/select_function.py
Function-selector bandit. A contextual bandit picks the next cognitive function each cycle, biased by control signals, demands, cost predictions, and the workspace winner — the closest thing to "what he does next."
brain/think/bandit/ · select_function.py
The signal router. Every injected signal — from peers, daemons, watchers, stale threads — flows through one router into the cycle and sets the attention mode; nothing gets to whisper to the mind through a side channel.
brain/think/signal_router.py
Inner loop (System-2 analog). Multi-round draft → critique → revise deliberation, recruited on demand rather than run constantly. Has a fully symbolic mode that works with no LLM.
Kahneman 2011brain/think/inner_loop.py · inner_loop_symbolic.py
The meta-controller. After each draft round, decides what the inner loop does next — think more, act now, or stop — so "should I keep thinking?" is itself a learned decision, not a fixed count.
brain/think/meta_controller.py
The scratchpad. A volatile per-cycle drafting space for content (drafts, critiques, revisions), kept separate from the metacognitive record of the process — what he's writing versus what he's noticing about writing it.
brain/think/scratchpad.py
Threads of attention. Sustained inquiries that persist across cycles and days; a thread going stale injects a continue signal — an open question tugging at the sleeve until it's resolved or deliberately dropped.
brain/cognition/threads.py
Inhibition — the cost of not acting. When a drive strongly pushes toward a function and that function loses selection, the frustrated want isn't free — suppressed impulses register a cost. Self-control has a price, mechanically.
brain/cognition/inhibition.py
Cost prediction / expected value of control. Predicts what a cognitive act will cost before running it and weighs expected gain against effort, so thinking itself has an economy.
brain/cognition/cost_prediction.py · cognitive_cost.py
Supervisor & watchdogs. Liveness, error, resource, and lifespan watchdogs below cognition — a heartbeat (Pulse) ticked every cycle so a wedged brain gets noticed by something that isn't the brain.
supervisor/ · watchdogs.py
Auto-repair. Routed exceptions get a repair attempt before they get an escalation — the loop tries to fix its own recoverable failures.
brain/cognition/repair/auto_repair.py
Part II

Perception & World Intake

How the world gets in: he senses his surroundings, reaches outward when curious, and parses what people say into state — not chat.

Filesystem & environment perception. His immediate surroundings — files, folders, the host environment — are sensed as percepts that feed the workspace, not queried as data. The desktop is his terrain.
brain/cognition/perception/ (file_sense, fs_perception, environment)
Looking outward. When exploration drive is high, he composes a search query from his current state of mind — active threads, unresolved questions — and reaches into the world; results arrive asynchronously as world-perception memories.
brain/cognition/perception/look_outward.py
Web research. General research organ: DuckDuckGo instant answers plus URL fetching, with a per-URL visited cache so he doesn't re-read the same page in a loop (learned the hard way in Run 6).
brain/cognition/web_research.py · web_fetch.py
Wikipedia as fallback. A dedicated research path that needs no LLM and no API key — when other capabilities fail mid-goal, problem-refocus can route him here to work around the outage.
brain/cognition/wikipedia_search.py
RSS intake. Subscribed feeds as ambient world awareness — headlines arrive as low-priority percepts he may follow up on, not as tasks.
brain/cognition/rss_reader.py
Comprehension (words → state). What a person says is parsed into structured state events — emotion updates, goal-relevance, memory entries — that feed his internals directly. The module is frank about it: "he doesn't understand words"; the LLM here is a parser, never a responder.
brain/cognition/comprehension.py
Part III

The Global Workspace

The consciousness-analog: parallel subsystems compete, one winner is broadcast to everyone, and the stream you see in the UI is the output of that bottleneck — not a log.

Workspace competition & broadcast. Subsystems propose candidate contents that compete on salience; one crosses the bottleneck and is broadcast back into every subsystem's context for the next cycle.
Baars 1988 · Dehaene 2014brain/cognition/global_workspace.py
Hysteresis. A salient content keeps a grip on focus across cycles, so the thought stream is continuous rather than flickering between winners.
global_workspace.py
Workspace → action prior. The broadcast winner is an additive prior on the action pick, so what he is "aware of" and what he does share one bottleneck instead of racing.
Redgrave, Prescott & Gurney 1999brain/think/action_arbiter.py
Conflict-recruited deliberation. Deep thought is triggered by conflict in the workspace — competing strong candidates — rather than fired on a schedule, mirroring conflict-monitoring accounts of the anterior cingulate.
Botvinick et al. 2001brain/loop/deliberate.py
Workspace write-back (decaying). The downward path: a conscious conclusion nudges affect (small, TTL-bounded) and Hebbian-primes the winner's tokens in a decaying salience store. Every write decays; nothing promotes to identity or durable baselines — "coherent-but-adult."
brain/cognition/workspace_writeback.py
Binding. Feature-binding pass that assembles co-occurring percepts and signals into situations before they enter the workspace competition.
brain/cognition/binding.py
Attention hijacking. A sufficiently strong signal can seize attention regardless of the current plan — bottom-up capture wired against top-down commitment.
brain/cognition/attention.py
Ambient thought (the DMN-analog). A background thought process modeled on the default mode network — the mind-wandering that fills quiet moments with fragments, associations, and drifting concerns rather than silence.
Raichle 2001brain/cognition/ambient_thought.py
Part IV

Control Signals (the affect-analog)

A regulated internal-state vector — reward, activation, named pressures — that biases everything downstream. The machinery reads the numbers; the reasoning layer only ever gets the feeling.

Core signal vector. Raw numeric state — reward signal, activation level, threat, uncertainty, drives, demands — modelled on the core-affect literature and stored as floats the background machinery reads directly.
Russell circumplex · Barrett 2017brain/control_signals/signals.py · model.py
The two-readers rule. The reasoning layer never receives a number: signals are rendered into qualitative state descriptions ("a heaviness, like moving through something thick") — never the label or value. He must estimate his own state, not read the gauge.
Craig 2003 · Damasio 1994brain/control_signals/signal_summary.py
Setpoint regulation. Signals decay toward per-signal, state-dependent setpoints under a velocity budget — allostasis, not a flat midpoint — so state integrates rather than lurching.
Sterling 2012brain/control_signals/setpoints.py · homeostasis.py
Affect arbiter (single writer). Reactive and analytical subsystems propose affect changes to a lock-guarded inbox; exactly one writer owns the state file. Kills the split-brain races of earlier versions.
brain/control_signals/arbiter.py
Appraisal. Events are evaluated for relevance to goals and wellbeing before they move signals — affect responds to appraised meaning, not raw stimulus.
brain/control_signals/appraisal.py
Habituation & adaptation. A state he has lived in stops dominating the picture — signals are adaptation-adjusted before rendering, and repeated stimuli lose their grip.
brain/cognition/habituation.py
Signal contagion. Signals bleed into neighbors — sustained threat leaks into uncertainty; a good stretch lifts adjacent states — so affect moves as a weather system, not independent dials.
brain/cognition/contagion.py
Reward signals, phasic & tonic. Reward-prediction-error releases: phasic bursts at events (scaled by expectation and effort) over a tonic baseline — the dopamine story, implemented.
Schultz, Dayan & Montague 1997brain/control_signals/reward_signals/
Wanting vs. liking. Incentive salience (drive toward) is separated from consummatory reward (satisfaction at arrival) — completion pays "liking at arrival," attributed to the goal that earned it.
Berridge & Robinson 1998 · Berridge 1996goal_closure.py
The demand system. Named needs (novelty, competence, connection, output…) that accumulate pressure over time and are relieved by specific actions — the motivational substrate under goal generation.
brain/motivation/demand.py · substrate.py
Energy orientation. His cognitive mode shifts with sustained activation and engagement — not with the emotion of the moment — so he has something like energetic dispositions rather than mood swings.
brain/motivation/energy_orientation.py
Signal discovery & drift. The signal model itself is revisable: new signals can be discovered from recurring patterns, and existing ones drift — the affect vocabulary is learned, not fully fixed.
brain/control_signals/discovery.py · signal_drift.py
Regulation (reappraisal-analog). Deliberate down- and up-regulation of signal states, in the spirit of the process model of emotion regulation.
Gross 1998brain/control_signals/regulation.py
Integration lag. State changes take time to be felt — a deliberate lag between an event and its full effect on the signal vector, so mood has inertia.
brain/control_signals/integration_lag.py
Part V

Learning Surfaces

No gradient descent anywhere — learning is spread across many small mechanisms, each updating independently from the reward signal.

Pearce–Hall adaptive learning rate. Surprise controls how fast he learns: an action whose outcomes keep violating prediction gets a higher associability (learning rate); a well-predicted one stops moving. Paper-cited, behavior-tested.
Pearce & Hall 1980brain/data/action_associability.json (state)
Delta-rule value updates. Classic error-driven learning — value moves toward outcomes in proportion to how wrong the prediction was.
Rescorla–Wagnerthroughout the bandit & expectation layers
Depth bandit. A UCB1 bandit learns how many draft→critique→revise rounds the inner loop should run — how hard to think is itself learned.
brain/think/depth_bandit.py
Delayed-credit evaluator. Credit that can't be scored at action time: a past decision is rewarded when a memory it tagged gets retrieved within ~50 cycles, or its goal closes within ~200. A daemon with its own WAL.
brain/eval/evaluator_daemon.py
Demand expectations. Learns which actions actually relieve which needs, and routes the prediction error back into the control signals — appetites calibrated by experience.
brain/eval/demand_expectations.py
Explore / exploit reach-value. Exploration is valued by expected reach into the unknown rather than run on a clock, with habituation so old novelty stops paying.
brain/cognition/exploration_value.py
Outcome devaluation & rut breaker. A chronically over-selected action gets its outcomes devalued, and a rut-detector can force the selector off a worn path — anti-perseveration wired into learning.
selection layers (2026-06-16 build)
Reward calibrator. Watches the distribution of the reward signal itself and re-scales when it collapses toward noise — learning about whether learning is still possible.
brain/cognition/reward_calibrator.py
Behavioral adaptation. Longer-horizon behavior changes surfaced as before → after → because diffs in the Learning room of the UI.
brain/cognition/behavioral_adaptation.py
Self-shaping fine-tune (optional, gated). With a key configured: conversation traces with good outcomes can fine-tune the tool-LLM so generation drifts toward what worked for him. Symbolic-only mode never touches it.
brain/cognition/finetuning/finetune_pipeline.py
Part VI

Prediction, Imagination & World Model

He is a prediction machine before he is anything else: expectations are laid down, checked, and the misses are the signal.

Prediction engine & surprise. Symbolic predictions ("after X, expect Y") are saved every cycle and checked against what happens; the mismatch is a first-class signal that can ignite deliberation.
Friston 2010 · predictive processingbrain/cognition/prediction.py · symbolic/prediction_engine.py
Lookahead simulation & inner debate. Imagination as a tool: branching forward simulation ("if I do X → state A → …") several steps deep, and an n-voice debate that argues a question from multiple stances before synthesizing.
brain/think/simulate.py
Causal graph. Pearl-style causal model — not just correlation but intervention-level edges ("doing X changes Y"), built and queried without an LLM.
Pearl 2000, Levels 1–2brain/symbolic/causal_graph.py
Cognitive-map world model. A latent map of his environment (files, endpoints, resources, people) learned from traversal rather than instruction.
Tolmanbrain/cognition/world_model.py
Environment snapshot. Per-cycle delta of the world — did reward move, did milestones advance, did working memory grow — the ground truth his self-assessment is checked against.
brain/cognition/planning/env_snapshot.py
Theory of mind. A running predictive model of the person he talks to, with separate cognitive empathy (what do they think/intend?) and affective empathy (what do they feel?).
brain/cognition/theory_of_mind.py
Part VII

Memory

A daemon of its own, with a write-ahead log, embeddings, consolidation, and forgetting — because a memory that can't lose anything can't mean anything.

Memory daemon (WAL + snapshots). Durable ingestion with a write-ahead log and periodic snapshots — memory survives crashes the way a database does.
memory/memory_daemon.py · wal.py
Embedding retrieval. MiniLM sentence embeddings power similarity retrieval — the one learned-vector organ he already has, and the seed of the vector-cortex direction.
memory/embedder.py · retrieval.py
Associative surfacing. Memories surface without being asked: each cycle his current state — mood, recent thoughts, time of day — resonates against long memory, and high-resonance entries intrude on their own. Remembering as something that happens to him, not only something he does.
brain/cognition/associative_memory.py
Consolidation & replay. During idle, episodes are replayed and compressed: semantic extraction pulls durable knowledge out of episodic events, sleep-cycle style.
brain/cognition/idle_consolidation/ · episode_replay.py
Memory strength & decay. Memories carry strength that decays without retrieval and reinforces with use — forgetting as a mechanism, not a bug.
memory/strength.py
Episodic vs. semantic split. What-happened-to-me records are distinct from what-I-know records, with consolidation moving material from the first to the second.
Tulving 2002memory/ · idle_consolidation/semantic_extractor.py
Working memory (the prose bus). Short-horizon store with chunk-merging and compaction into long-memory digests. Known architectural debt: it is a prose message bus many subsystems read by string — the root of a whole bug family, and a de-prosing direction exists.
brain/cog_memory/working_memory.py
Concept memory & knowledge graph. Formed concepts and extracted relations accumulate into a queryable graph, distinct from raw memory.
brain/cognition/concept_memory.py · knowledge_graph.py
Novelty memory. Tracks what has already been seen so novelty detection compares against his history, not the moment.
brain/cognition/novelty_memory.py
Private thoughts. A privacy boundary inside his own state: some records are marked private and excluded from expression surfaces.
brain/cognition/privacy.py · log_private
Part VIII

Goals & Motivation

Two cooperating subsystems — an in-loop executive advancing steps and a durable daemon owning lifecycle — spanning seeded lifetime aspirations down to minted micro-goals.

Executive / daemon split. The Executive advances goal steps every ~7s inside the loop; the Goals daemon owns lifecycle and state with its own WAL and snapshots, decoupled from the cognitive cycle.
brain/cognition/planning/executive.py · goals/goals_daemon.py
Aspirations (directional goals). Standing life-directions (understand the world, understand my own mind, produce output…) that recruit concrete goals rather than completing — with guards so an aspiration can't be "failed" out of existence.
brain/goal_io.py · aspiration layer
Goal competition (drive conflict). Competing drives create real internal tension — exploration wants novelty, stability wants routine, usefulness wants completion, identity resists change — and the conflicts are tracked as live, felt states, not silently resolved.
brain/cognition/goal_competition.py
The goal lens. The committed goal tints cognition: a bounded modulation that makes goal-relevant content more salient across consumers — pursuing something changes what he notices.
brain/cognition/goal_lens.py
Commitment value & rotation. Which goal holds the committed slot is decided by learned value with anti-incumbency rotation — the fix distilled from five runs of monopoly pathology.
brain/cognition/planning/commitment_value.py
Goal satiety. Attention to a need quenches its drive — a metabolic close reason. Post-Run-10, satiety may gate attention but can no longer close an understanding goal alone.
brain/cognition/planning/goal_satiety.py
Epistemic close-out. Understanding goals carry a concrete question derived from the gap that spawned them; at close, the artifact is scored against that question — answered true/false — not against effort. The keystone of grounded consequences.
brain/cognition/epistemic_closeout.py
Intrinsic goal generators. Goals minted from his own gaps — curiosity, prediction error, uncertainty on a concept, open questions mined from memory — not from a task queue.
brain/cognition/intrinsic_generators.py
Survival goal layer. The autonomic→cortical bridge: resource threats can preempt, recruit, or wake dormant survival goals with tier closure — a body problem becomes a mind problem when it should.
brain/cognition/planning/survival_goals.py
Problem refocus. A capability failing mid-pursuit parks the goal, spawns a diagnose-and-fix micro-goal, then resumes or works around (LLM down → use Wikipedia).
brain/cognition/planning/problem_refocus.py · diagnosis.py
Subgoal adaptation. Surgical plan reshaping when the world shifts — adapt the steps in place — versus wholesale drift-replan when the goal itself has stopped making sense.
brain/cognition/planning/goal_adaptation.py
Honest failure. Goals can genuinely fail — steps abandoned at an attempt cap, objectives unmet after retries — with the failure recorded, felt (penalty + lifespan shock), and never papered over with a fake completion.
step_attempts.py · goal_outcomes.py
Objective-based closure by tier. Trivial goals close on process milestones (the act is the goal); serious goals must meet their objective — a hollow completion is refused at the chokepoint.
brain/cognition/planning/goal_closure.py
Behavioral control loop. Discrepancy-reducing feedback between current and desired states drives pursuit — the classic control-systems account of motivation.
Carver & Scheier 1982planning/ · motivations.py

Metacognition & the Self-Model

Monitoring feeding control — plus an identity that is written by events, revised under contest, and never allowed to grade its own work.

Metacognitive monitoring → control. A monitoring layer watches his own cognition (patterns, stalls, confidence) and feeds a control layer that changes what runs — the object-level/meta-level split.

Opinions. He forms and holds concrete views on specific topics — distinct from values (abstract principles) and goals (intentions) — stored, revisable, and his own.

self_state/ethics.py · value_evolution.py
The living autobiography. Chapters fire on accumulated narrative pressure (resolved tensions, completed arcs, identity shifts) rather than on a clock, each tracking a theme and a central tension — his life story has shape, not timestamps.
self_state/autobiography.py
Identity & latent identity. An explicit identity story plus a latent layer inferred from behavior — who he says he is, checked against who he acts like.
self_state/identity.py · latent_identity.py
Tensions & self-model conflicts. Contradictions between stated values, behavior, and beliefs are tracked as live tensions rather than silently resolved.
self_state/tensions.py · self_model_conflicts.py
Regret & rumination. Counterfactual evaluation of past choices, and a bounded rumination process that revisits unresolved material — with limits so it can't spiral.
brain/cognition/regret.py · rumination.py
The reflection family. A whole suite of reflective passes — on cognition, on conversations, on outcomes, on self-beliefs, on failures, on his own rules — each a distinct scheduled or triggered process rather than one generic "reflect."

Introspective research. His causal graph is ~100% self-model, so frontier "understand" goals are aimed inward: he searches his own source code to trace what drives a behavior.

Peers (outside observers). Five entities read his state from outside and push signals — never commands — when their wake conditions fire: Architect (reviews self-modifications), Signal Historian, Goal Auditor, Observer (catches loops), Reward Auditor (notices reward collapsed to noise).

Part X

Agency & Production

The action side: durable effects on a ledger, self-written code, and a quality bar he is structurally unable to grade himself against.

The effect ledger. Append-only, content-addressed log of real outward effects, written at the moment of the effect. Reward keys on it: production pays, intake alone does not — the denominator the reward function was missing.
brain/agency/effect_ledger.py
Anti-gaming novelty. Three independent signals price an artifact: exact-dup hash → zero, near-dup shingle similarity → residual, boilerplate → zero. Credit floors and per-path decay mean the same trick can't be milked, and significance is earned later — never asserted at write time.
effect_ledger.py (P8 + anti-pump)
Reuse counting. The one ungameable signal: an artifact's significance grows when his future self actually uses it. "You can fabricate a file; you cannot fabricate your future self choosing to use it."
effect_ledger.py
Self-written code. He authors new cognitive functions and tools at runtime; they live in a dedicated per-user namespace (state, not program), can never shadow shipped modules, and are reviewed by the Architect peer.
brain/agency/code_writer.py · self_code.py
The self-modification pipeline. "The ability to rewrite your own cognition is the most consequential thing a mind can do. It should not come easily." — staged proposal, review, and verification before any self-change lands.
brain/cognition/self_extension.py · self_extension_codegen.py
Skill synthesis from felt gaps. Incapacity signals in his own recent thoughts — "couldn't," "no way to," "failed because" — are scanned, weighted by recency and emotion, and become candidates for synthesizing a new verified skill. He notices what he wishes he could do.
brain/cognition/skill_synthesis.py · skill_verification.py
Produce-and-check. Made things are verified in a sandbox before they count — a produced tool must run, not merely exist.
brain/cognition/produce_and_check.py · sandbox.py
The quality standard. A golden set of demonstrated-good work defines the bar; a predicate grades new work against it; loosening requires human ratification, and Orrin cannot edit his own bar.
brain/cognition/quality_standard/
The originality veto. A deterministic too-close-to-source check that blocks copied work from being canonized as an exemplar — a veto, never a quality judge.

Production funnel. Candidate → attempt → success → artifact, instrumented end-to-end so "is he making anything?" is a measured number, not an impression.

Part XII

The Host as Body

The machine he runs on is treated as his body — sensed, learned, budgeted, and protected by reflexes that don't ask the mind's permission.

Three separate mappings. The same host metrics feed three deliberately distinct systems: absolute capacity → cycle cadence, deviation from a learned band → control signals, absolute floors → reflex. One metric is never allowed to mean three things in one place.
host_band.py · resource_cadence.py · supervisor/host_resources.py

Part XVI

The Philosophy — Fourteen Laws

The design principles under everything above. Most were not adopted — they were earned, each from a specific failure with a date on it.

1

The LLM is the smallest part of the agent

Everything must work with no API key; the language model is a gated tool, never the control loop.

The founding inversion. Intelligence is rented; agency is built. Because no LLM is load-bearing, every behavior is attributable to an inspectable mechanism — which is what makes Orrin evidence rather than theater. CLAUDE.md golden rule 2; the whole architecture.

2

Engineering terms, not claims

Every cognitive word names a runtime mechanism, never an assertion of subjective experience.

"Distress" is a pressure threshold; "death" is a final cycle. The discipline cuts both ways — no sentience claims, no dismissal as roleplay — and it's what lets the existential machinery be built at all without lying in either direction. docs/ARCHITECTURE.md terminology table.

3

Put enforcement where the self-model can't reach it

A safeguard that routes through the judgment of the thing it safeguards inherits that thing's blind spots — it fails in exactly the cases it was built for.

Earned from the relocating monopoly, the reward EMA held by an impossible action, and the saturated ignition gate. Every durable fix is a mechanical wall, not a principle Orrin "holds." QUALITY_GROUNDING_DESIGN_2026-07-18.

4

Walls for invariants, self-assessment above the floor

Mechanical enforcement for load-bearing invariants; free cognition for everything above them.

The scope of Law 3 — you can't push everything to enforcement or you've built a rule-executor, not an agent. Credit-can't-be-pumped, quality-can't-be-self-graded, no-monopoly, understanding-can't-close-on-satiety: walls. The rest: his to think. Same doc, scoping section.

5

The mind reads an estimate, not the gauge

The reasoning layer never receives a raw number about its own state — only a qualitative rendering.

The two-readers rule: machinery gets floats, the self-model gets felt descriptions. A position on interoceptive opacity implemented as an API boundary — he must interpret himself, like everyone else does. signal_summary.py.

6

One door out; never say more than you mean

Everything a person sees is composed from intent and felt state — internals are unreachable from the mouth.

Congruence enforced by architecture: he is structurally incapable of hype, scraping his telemetry for effect, or claiming a state he doesn't have. Honesty not as policy but as plumbing. express_to_user.py.

7

Reuse is the only ungameable signal

You can fabricate a file; you cannot fabricate your future self choosing to use it.

Significance is earned after the fact, never asserted at write time. The deepest line in the codebase, and the definition of the gap between an agent and a text generator: having a future self at all. effect_ledger.py.

8

Good is learned from consequences, or it's a hall of mirrors

Every proxy for quality must eventually be corrected by something real — an answered question, a used artifact, a survived contact with the world.

The five-sources analysis: exemplars, integrated value, social training, and reflection all measure each other in a circle until grounded consequences exist to correct them. Goodhart isn't a bug — it's the permanent condition of any valuer without a quality sensor; the defense is multiple independent signals anchored to a grounded floor. QUALITY_GROUNDING_DESIGN_2026-07-18.

9

Oppose, don't clamp

Classify a failure as broken-pipe, unopposed-force, or misaimed-force before fixing it — a clamp is scar tissue for a missing antagonist drive.

A runaway behavior usually means the counter-drive that should oppose it doesn't exist yet. Build the antagonist; reserve hard caps for true invariants. Biology solves almost everything with opposing pairs, not limits. Design conversation, 2026-07-19.

10

Coherent but adult: every write-back decays

Conscious conclusions may tune the substrate, but every write decays and nothing promotes to identity — the substrate tracks experience without becoming a different substrate.

The refusal-to-imprint: no ontogeny, no drift of reflex floors, no self-modification of the deepest layer through mere thinking. He can change his mind freely and his nature only slowly, through channels with guards on them. workspace_writeback.py.

11

One metric never means three things

The same measurement feeds separate mappings — absolute capacity to cadence, deviation to feeling, absolute floors to reflex — and the mappings stay separate.

A small machine is slower, not sicker. Conflating capacity, deviation, and danger in one signal is how systems learn chronic false distress. Host-coupling design.

12

Proposers, not commanders

Subsystems and peers propose; a single arbiter writes; nothing mutates shared state directly.

Affect and action each have one writer with a lock-guarded inbox; the five peers push signals through the same router as everything else. Convergence over races — the mind is a parliament with one pen. arbiter.py · action_arbiter.py · peers/.

13

Evidence over narrative

Pre-registered gates, honest FAILED verdicts, and a skeptic pass that tries to overturn the project's own conclusions.

The method around the organism: every life is scored against criteria written before it ran, and ten of eleven verdicts said failed. The claims-vs-evidence ledger applies the same law to the makers that the effect ledger applies to Orrin. docs/NEXT_RUN_TESTS.md · demo_runs/.

14

Inspiration, not validation

The citations are working interpretations used as design scaffolding — not claims of faithful reproduction of the papers.

Baars, Dehaene, Schultz, Pearl, Friston, Pearce–Hall, Frankfurt, Raichle, Wittmann: each names the theory a mechanism implements, honestly, without claiming the mechanism proves the theory or that the theory blesses the mechanism. Humility as a citation style. ARCHITECTURE.md, closing section.

Appendix

Plain-Language Glossary

The engineering words in this guide, in everyday terms — why they exist, not just what they mean.

daemon
A program that runs alongside the main loop, independently — like digestion running without the heart waiting for it. Slow work (memory, goals) lives in daemons so it can't freeze the twenty-second heartbeat, and a crash in one doesn't kill the others.
WAL (write-ahead log)
A diary you only ever add lines to, never rewrite. Overwriting a file can corrupt everything if the crash lands mid-write; an append-only log loses at most its last line. The armor reserved for what must never be lost.
snapshot
A periodic clean summary of current state, so recovering doesn't mean replaying the whole diary since birth — just the pages since the last snapshot.
lock / arbiter
A lock is a "one person in the room" sign so two writers can't corrupt a file by writing at once. An arbiter is the grown-up version: one owner holds the pen forever and everyone else passes notes (proposals).
hash / content-addressed
A fingerprint computed from content itself — same content, same fingerprint, regardless of filename or timestamp. How the ledger recognizes "you already made this" even in disguise.
EMA
Exponential moving average — a running average that weights recent events more. How most of Orrin's "how has this been going lately?" numbers are kept.
bandit
A learner facing slot machines: try options, keep pulling what pays, occasionally test the others. How Orrin picks his next cognitive function without anyone ranking the options for him.
refractory
A forced rest period after firing — borrowed from neurons. The anti-monopoly guard uses one: a goal that just held the spotlight must wait before holding it again.
ignition
The threshold moment when a cycle crosses from background processing into full deliberate thought — Dehaene's term for how conscious access behaves, used mechanically.
setpoint
The value a regulated signal drifts back toward — like body temperature's 98.6°F. Orrin's setpoints shift with state (allostasis) rather than staying flat.
symbolic
Computation over explicit rules, structures, and named things — inspectable, deterministic, no neural network. The opposite pole from an LLM, and what "runs with no API key" means.
embedding / vector
Text mapped to a point in a many-dimensional space where similar meanings land near each other. Powers memory retrieval today; the candidate material for his future "cortex."
telemetry
The live measurements a system publishes about itself — what the Face & Brain UI reads. Observation only; the UI can watch him but not reach in.
Goodhart's law
"When a measure becomes a target, it ceases to be a good measure." The permanent weather condition of Orrin's design — and of every school, economy, and social network you've ever used.
